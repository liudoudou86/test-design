# 行业标准与大纲模板

本文档提供不同开发模式和行业要求的测试大纲模板与参考标准。

## A. 大纲结构模板

### 模板1：敏捷迭代（Sprint）测试大纲（快速、聚焦） <a id="agile"></a>
适用于2-4周冲刺周期内的功能测试。
```markdown
### [功能名称] - 测试要点 (Sprint [编号])
**一、本次迭代测试范围**
*   **包含**：[列出本Sprint承诺完成的用户故事/功能点]
*   **不包含**：[明确说明不在本次测试范围的关联功能]

**二、测试重点与策略**
1.  **核心功能验证**：针对每个用户故事的"验收标准"进行逐条测试。
2.  **新功能回归**：对新功能进行快速的正向、反向测试。
3.  **核心路径冒烟**：确保主业务流程在新代码合并后仍然畅通。
4.  **缺陷验证**：验证与本迭代相关的已修复缺陷。

**三、测试环境与数据**
*   **环境**：SIT（系统集成测试）环境。
*   **数据**：使用预制测试账户和测试数据。

**四、风险与依赖**
*   后端API延迟交付。
*   第三方服务接口不稳定。
```

### 模板2：版本发布测试大纲（完整、全量） <a id="release"></a>
适用于正式版本发布前的全量测试。
```markdown
### [版本号] 发布测试大纲
**一、测试范围**
*   **新功能**：[列出本次发布新增功能]
*   **回归范围**：[列出需回归的已有功能模块]
*   **不包含**：[明确说明不在本次测试范围的内容]

**二、测试类型与轮次**
1.  **第一轮 - 冒烟测试**：核心路径 + P0用例，阻塞缺陷为零
2.  **第二轮 - 功能测试**：全量功能用例执行
3.  **第三轮 - 集成测试**：跨模块业务流程、数据流转
4.  **第四轮 - 回归测试**：修复验证 + 受影响模块回归
5.  **第五轮 - 验收测试**：对标验收标准逐条通过

**三、测试策略**
*   **功能测试**：正反场景全覆盖
*   **UI/UX测试**：设计稿对照
*   **性能测试**：基准测试 + 负载测试
*   **安全测试**：OWASP Top 10 检查
*   **兼容性测试**：目标浏览器/设备矩阵

**四、准入/准出标准**
*   **准入**：开发自测通过，代码 Review 完成，提测邮件发出
*   **准出**：P0/P1用例通过率100%，P2/P3通过率≥95%，无致命/严重缺陷遗留

**五、风险与应对**
*   [风险] → [应对措施]
```

### 模板3：安全关键系统测试大纲（严格、合规） <a id="safety-critical"></a>
适用于金融、医疗、自动驾驶等合规性要求高的系统。
```markdown
### [系统名称] 安全关键测试大纲
**一、适用标准**
*   [ ] ISO 26262（汽车功能安全）/ IEC 62304（医疗软件）/ PCI-DSS（支付行业）
*   [ ] 等保2.0 / GDPR / HIPAA

**二、强制测试类型**
1.  **安全测试**：渗透测试、SQL注入、XSS、CSRF、越权、会话管理
2.  **可靠性测试**：故障注入、恢复测试、冗余切换
3.  **合规性测试**：数据加密、审计日志、隐私保护
4.  **性能测试**：峰值负载、持续稳定性（≥72小时）

**三、测试级别**
*   **单元测试**：覆盖率 ≥ 90%（关键模块100%）
*   **集成测试**：接口契约验证
*   **系统测试**：全功能端到端
*   **验收测试**：三方审计 / 监管验收

**四、缺陷分级**
| 级别 | 定义 | 响应时间 |
|------|------|----------|
| 致命 | 安全漏洞/数据丢失 | 立即修复，≤4小时 |
| 严重 | 核心功能不可用 | ≤24小时 |
| 一般 | 非核心功能异常 | ≤5工作日 |
| 建议 | 体验优化 | 下一版本 |
```

## B. 参考标准

### ISTQB 测试计划模板 <a id="istqb"></a>

参考 ISTQB 标准，测试计划应包含以下要素：

| 要素 | 说明 |
|------|------|
| 测试范围 | In Scope / Out Scope |
| 测试策略 | 测试级别、测试类型、测试方法 |
| 资源计划 | 人员、环境、工具 |
| 进度计划 | 里程碑、各轮次时间安排 |
| 风险分析 | 项目风险、产品风险及应对 |
| 准入/准出标准 | 进入和退出每个测试阶段的条件 |
| 交付物 | 测试用例、测试报告、缺陷报告 |
