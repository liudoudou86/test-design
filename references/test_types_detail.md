# 测试类型详解

本文档定义了在生成测试大纲时应考虑的各种测试类型及其核心要素。

## 1. 功能测试
*   **测试目标**：验证软件功能是否符合需求规格说明，确保每个功能能按预期工作。
*   **典型场景**：
    *   **正向流程**：验证核心业务流程（如：用户注册 -> 登录 -> 完成购买 -> 登出）。
    *   **异常流程**：验证系统对无效输入、中断、错误操作的处理（如：支付中断后订单状态、输入超长用户名）。
    *   **边界值分析**：针对输入条件的边界进行测试（如：允许上传的文件大小为1-10MB，则测试0.99MB, 1MB, 10MB, 10.01MB）。
*   **验证重点**：功能正确性、数据准确性、错误处理能力。

## 2. 用户界面(UI)/用户体验(UX)测试
*   **测试目标**：确保界面设计符合规范，用户交互符合直觉，体验流畅。
*   **典型场景**：
    *   **视觉与布局**：检查元素对齐、颜色、字体、间距是否符合设计稿。
    *   **交互反馈**：验证按钮点击、表单提交、页面跳转是否有正确的视觉或提示反馈。
    *   **易用性**：检查导航是否清晰，关键信息是否突出，操作步骤是否繁琐。
*   **验证重点**：一致性、易用性、可访问性（基础）。

## 3. 兼容性测试
*   **测试目标**：确保软件在不同环境下都能正常工作。
*   **典型场景**：
    *   **跨浏览器测试**：在 Chrome, Firefox, Safari, Edge 等主流浏览器的最新及前1-2个版本进行测试。
    *   **跨平台/设备测试**：对于Web端，测试不同操作系统（Windows, macOS）。对于移动端，测试不同品牌、型号、屏幕尺寸、分辨率的真机或模拟器。
    *   **响应式设计**：测试页面在不同屏幕宽度下的布局适应性。
*   **验证重点**：布局/功能无错位、无崩溃，核心流程在所有目标环境畅通。

## 4. 性能测试
*   **测试目标**：评估系统在不同负载下的响应速度、稳定性和资源消耗。
*   **典型场景**：
    *   **负载测试**：在典型预期用户并发下，验证系统的性能指标（如：100用户同时浏览商品页）。
    *   **压力测试**：在极限或超过极限的负载下，验证系统是否会崩溃或数据出错（如：1000用户同时秒杀）。
    *   **基准测试**：对某个关键操作进行单用户测试，建立性能基准（如：页面加载时间<2秒）。
*   **验证重点**：响应时间、吞吐量(TPS/QPS)、错误率、CPU/内存使用率。

## 5. 安全测试
*   **测试目标**：发现系统中可能被恶意利用的安全漏洞，保护数据和系统。
*   **典型场景**：
    *   **注入攻击**：测试SQL注入、命令注入等。
    *   **身份验证与授权**：测试弱密码、会话管理、越权访问（垂直/水平）。
    *   **数据安全**：验证敏感信息（密码、个人身份证号）是否在前端暴露、传输中是否加密。
*   **验证重点**：机密性、完整性、可用性、权限控制。

## 6. 无障碍测试
*   **测试目标**：确保软件可以被残障人士（如视障、听障）使用，通常遵循WCAG标准。
*   **典型场景**：
    *   **屏幕阅读器兼容**：验证所有功能和信息可以通过屏幕阅读器获取。
    *   **键盘导航**：验证所有功能可以通过键盘（Tab, Enter, 方向键）完成操作。
    *   **色彩对比度**：验证文本与背景的对比度满足最低要求。
*   **验证重点**：可感知性、可操作性、可理解性、兼容性。

---
*提示：在生成大纲时，应根据产品特性（如金融App需侧重安全，视频网站需侧重性能）选择上述类型的子集并进行优先级排序。*
